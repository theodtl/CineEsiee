GUIDE ORAL - PROJET CINEESIEE
=============================

Objectif du document
--------------------

Ce document sert a preparer l'oral du projet CineEsiee.
Il explique :

- la logique generale du programme ;
- le role de chaque fichier ;
- les structures de donnees ;
- l'enchainement des fonctions ;
- le role de chaque fonction importante ;
- les questions possibles du professeur et les reponses attendues.


1. Idee generale du projet
--------------------------

Le programme simule une petite base de donnees cinematographique.

Il gere :

- des acteurs ;
- des realisateurs ;
- des films ;
- des cinemas relies par des routes.

Les donnees ne sont pas codees en dur dans main.c.
Elles sont lues au demarrage depuis le fichier donnees.csv.

Le programme propose ensuite un menu console pour :

1. lister les films ;
2. lister les realisateurs ;
3. rechercher un film par titre ;
4. quitter ;
5. lister les acteurs ;
6. calculer un trajet optimal entre deux cinemas ;
7. afficher la filmographie d'un acteur ;
8. afficher la filmographie d'un realisateur.


2. Organisation des fichiers
----------------------------

main.c
    C'est le fichier principal.
    Il contient la fonction main, le menu, les saisies utilisateur et les appels
    aux fonctions des autres modules.

personne.h / personne.c
    Ces fichiers definissent et manipulent les personnes :
    Date, Personne, Acteur, Realisateur.
    Ils gerent aussi les listes chainees d'acteurs et de realisateurs.

film.h / film.c
    Ces fichiers definissent et manipulent les films.
    Un film contient un titre, une annee, une duree, un genre, un realisateur
    et jusqu'a 3 acteurs.

cinema.h / cinema.c
    Ces fichiers definissent le graphe de cinemas.
    Ils gerent les cinemas, les routes entre cinemas et l'algorithme de
    Dijkstra pour trouver le plus court chemin.

fichier.h / fichier.c
    Ces fichiers chargent les donnees depuis donnees.csv.
    Ils transforment les lignes du CSV en acteurs, realisateurs, films,
    cinemas et routes.

donnees.csv
    C'est le fichier de donnees.
    Il contient les acteurs, realisateurs, films, cinemas et routes.

README.txt
    C'est une documentation courte pour compiler et comprendre rapidement
    le projet.


3. Relations entre les fichiers
-------------------------------

main.c inclut :

    #include "cinema.h"
    #include "fichier.h"
    #include "film.h"
    #include "personne.h"

Cela permet au main d'utiliser les structures et fonctions des autres modules.

film.h inclut personne.h car un Film contient :

    Realisateur *realisateur;
    Acteur *acteurs[NB_ACTEURS_MAX];

Donc film.h doit connaitre les types Acteur et Realisateur.

fichier.h inclut cinema.h, film.h et personne.h car le chargement CSV doit
remplir toutes les parties de la base.

cinema.h est independant des films et des personnes. Il gere seulement le
graphe de cinemas.


4. Structures principales
-------------------------

4.1 Date

Dans personne.h :

    typedef struct
    {
        int jour;
        int mois;
        int annee;
    } Date;

Elle sert a stocker une date de naissance.


4.2 Personne

Dans personne.h :

    typedef struct
    {
        char nom[TEXTE_COURT];
        char prenom[TEXTE_COURT];
        char nationalite[TEXTE_COURT];
        Date naissance;
    } Personne;

Une Personne contient les informations communes a un acteur et a un
realisateur.

Cela evite de dupliquer du code entre Acteur et Realisateur.


4.3 Acteur et Realisateur

Dans personne.h :

    typedef struct
    {
        Personne infos;
    } Acteur;

    typedef struct
    {
        Personne infos;
    } Realisateur;

Un acteur et un realisateur sont deux types differents, mais ils contiennent
tous les deux une Personne.


4.4 Listes chainees d'acteurs et de realisateurs

Dans personne.h :

    typedef struct NoeudActeur
    {
        Acteur acteur;
        struct NoeudActeur *suivant;
    } NoeudActeur;

    typedef struct NoeudRealisateur
    {
        Realisateur realisateur;
        struct NoeudRealisateur *suivant;
    } NoeudRealisateur;

Chaque noeud contient une donnee et un pointeur vers le noeud suivant.
Cela permet d'avoir un nombre variable d'acteurs ou de realisateurs.


4.5 Genre

Dans film.h :

    typedef enum
    {
        ACTION = 1,
        HORREUR,
        COMEDIE,
        DOCUMENTAIRE,
        POLICIER,
        DRAME,
        ANIMATION,
        SCIENCE_FICTION
    } Genre;

Un enum permet de limiter les genres possibles.
Cela evite d'ecrire n'importe quelle chaine de caracteres dans un film.


4.6 Film

Dans film.h :

    typedef struct Film
    {
        char titre[TITRE_MAX];
        int annee;
        int duree;
        Genre genre;
        Realisateur *realisateur;
        Acteur *acteurs[NB_ACTEURS_MAX];
        int nb_acteurs;
        struct Film *suivant;
    } Film;

Un Film contient :

- son titre ;
- son annee ;
- sa duree en minutes ;
- son genre ;
- un pointeur vers un realisateur existant ;
- un tableau de pointeurs vers 1 a 3 acteurs existants ;
- un pointeur vers le film suivant.

Point important pour l'oral :
    Le film ne copie pas les informations du realisateur ni des acteurs.
    Il garde seulement des pointeurs vers eux.
    Cela evite la duplication des donnees.

Exemple :
    David Fincher est stocke une seule fois dans la liste des realisateurs.
    Fight Club, Seven et The Game pointent tous vers ce meme realisateur.


4.7 Cinema, Arete et GrapheCinema

Dans cinema.h :

    typedef struct Arete
    {
        Cinema *destination;
        int distance;
        struct Arete *suivant;
    } Arete;

Une Arete represente une route entre deux cinemas.
Elle contient :

- le cinema destination ;
- la distance ;
- la route suivante.

    struct Cinema
    {
        int id;
        char nom[CINEMA_NOM_MAX];
        Arete *voisins;
        Cinema *suivant;
    };

Un Cinema contient :

- un identifiant ;
- un nom ;
- une liste chainee de routes vers ses voisins ;
- un pointeur vers le cinema suivant.

    typedef struct
    {
        Cinema *cinemas;
    } GrapheCinema;

Le graphe contient la liste chainee des cinemas.


5. Chargement depuis donnees.csv
--------------------------------

Le fichier donnees.csv contient plusieurs types de lignes :

    ACTEUR;nom;prenom;jour;mois;annee;nationalite
    REALISATEUR;nom;prenom;jour;mois;annee;nationalite
    FILM;titre;annee;duree;genre;nom_realisateur;nom_acteur1;nom_acteur2;nom_acteur3
    CINEMA;id;nom
    ROUTE;id_depart;id_arrivee;distance

Le chargement est fait par la fonction charger_base dans fichier.c.
Quand un film est ajoute ou supprime depuis le menu, la fonction
sauvegarder_base reecrit donnees.csv.

Ordre important :

1. Charger les acteurs.
2. Charger les realisateurs.
3. Charger les films.
4. Charger les cinemas.
5. Charger les routes.

Les films doivent venir apres les acteurs et realisateurs, car le film doit
retrouver les pointeurs vers des personnes deja creees.

Les routes doivent venir apres les cinemas, car une route relie deux cinemas
qui doivent deja exister.


6. Deroulement du programme
---------------------------

Etape 1 : main commence

Dans main.c :

    NoeudActeur *acteurs = NULL;
    NoeudRealisateur *realisateurs = NULL;
    Film *films = NULL;
    GrapheCinema graphe;

Ces variables representent la base de donnees en memoire.

Etape 2 : initialisation du graphe

    initialiser_graphe(&graphe);

Cette fonction met la liste des cinemas a NULL.

Etape 3 : chargement du fichier CSV

    charger_base("donnees.csv", &films, &acteurs, &realisateurs, &graphe);

Cette fonction remplit les listes chainees et le graphe.

Etape 4 : boucle du menu

    while (choix != 4)

Tant que l'utilisateur ne choisit pas Quitter, le programme affiche le menu,
lit le choix, puis execute l'action correspondante avec un switch.

Etape 5 : liberation memoire

A la fin, le programme appelle :

    liberer_films(films);
    liberer_acteurs(acteurs);
    liberer_realisateurs(realisateurs);
    liberer_graphe(&graphe);

Cela libere la memoire allouee avec malloc.


7. Role des fonctions de main.c
-------------------------------

vider_fin_ligne
    Supprime le \n laisse par fgets a la fin d'une chaine.
    Sans cette fonction, une recherche de titre pourrait echouer car la chaine
    contiendrait un retour a la ligne invisible.

lire_choix
    Lit le choix du menu avec fgets, puis convertit en entier avec sscanf.
    C'est plus propre que scanf car cela evite certains problemes avec le
    buffer clavier.

lire_entier
    Lit un entier avec un message personnalise.
    Elle est utilisee pour les ids de cinemas.

pause_console
    Attend que l'utilisateur appuie sur Entree.
    Cela evite que le menu revienne trop vite apres un affichage.

rechercher_film_menu
    Demande un titre a l'utilisateur.
    Appelle chercher_film.
    Si le film existe, appelle afficher_film.

filmographie_acteur_menu
    Demande le nom d'un acteur.
    Cherche l'acteur dans la liste.
    Appelle afficher_filmographie_acteur.

filmographie_realisateur_menu
    Demande le nom d'un realisateur.
    Cherche le realisateur dans la liste.
    Appelle afficher_filmographie_realisateur.

trajet_menu
    Affiche les cinemas.
    Demande un cinema de depart et un cinema d'arrivee.
    Appelle optimiser_trajet.

afficher_menu
    Affiche les choix disponibles.

main
    Fonction principale :
    - declare les listes ;
    - charge donnees.csv ;
    - gere le menu ;
    - libere la memoire.


8. Role des fonctions de personne.c
-----------------------------------

copier
    Fonction interne static.
    Copie une chaine avec strncpy et ajoute '\0' a la fin.
    Elle evite de depasser la taille des tableaux de caracteres.

creer_date
    Cree et retourne une Date.

creer_personne
    Cree une Personne avec nom, prenom, date de naissance et nationalite.

creer_acteur
    Cree un Acteur a partir des informations donnees.
    En interne, elle appelle creer_personne.

creer_realisateur
    Cree un Realisateur a partir des informations donnees.
    En interne, elle appelle creer_personne.

ajouter_acteur
    Alloue un nouveau NoeudActeur avec malloc.
    Stocke l'acteur dans le noeud.
    Ajoute ce noeud en tete de liste.
    Retourne la nouvelle tete de liste.

ajouter_realisateur
    Meme principe que ajouter_acteur, mais pour les realisateurs.

chercher_acteur
    Parcourt la liste chainee d'acteurs.
    Compare les noms avec strcmp.
    Retourne l'adresse de l'acteur trouve ou NULL.

chercher_realisateur
    Meme principe que chercher_acteur, mais pour les realisateurs.

afficher_acteur
    Affiche les informations d'un acteur.

afficher_realisateur
    Affiche une fiche realisateur.

lister_acteurs
    Parcourt la liste et affiche chaque acteur.

lister_realisateurs
    Parcourt la liste et affiche chaque realisateur.

liberer_acteurs
    Parcourt la liste des acteurs et libere chaque noeud.

liberer_realisateurs
    Parcourt la liste des realisateurs et libere chaque noeud.


9. Role des fonctions de film.c
-------------------------------

copier
    Fonction interne static.
    Copie une chaine dans un tableau de caracteres.

comparer_sans_casse
    Compare deux chaines sans tenir compte des majuscules/minuscules.
    Cela permet de trouver Avatar meme si l'utilisateur tape avatar.

genre_en_texte
    Transforme une valeur de l'enum Genre en texte affichable.
    Exemple : SCIENCE_FICTION devient "Science-Fiction".

creer_film
    Cree une variable Film.
    Initialise le titre, l'annee, la duree, le genre, le realisateur.
    Initialise aussi le tableau d'acteurs a NULL et nb_acteurs a 0.

ajouter_acteur_film
    Ajoute un pointeur vers un acteur dans le tableau acteurs du film.
    Verifie qu'il reste de la place avec NB_ACTEURS_MAX.
    Le film peut donc avoir maximum 3 acteurs principaux.

ajouter_film
    Alloue un nouveau Film avec malloc.
    Copie le film recu.
    Ajoute le film en tete de liste chainee.
    Retourne la nouvelle tete de liste.

supprimer_film
    Cherche un film par son titre.
    Retire le noeud de la liste chainee.
    Libere le noeud du film avec free.

chercher_film
    Parcourt la liste des films.
    Compare le titre demande avec le titre du film.
    La comparaison ignore les majuscules/minuscules.
    Retourne le film trouve ou NULL.

afficher_film
    Affiche la fiche complete d'un film :
    - titre ;
    - annee ;
    - realisateur ;
    - acteurs ;
    - duree ;
    - genre.

lister_films
    Parcourt la liste chainee de films et appelle afficher_film pour chacun.

afficher_filmographie_acteur
    Parcourt tous les films.
    Pour chaque film, parcourt les acteurs du film.
    Si l'acteur est trouve, affiche le titre du film.

    Point important :
        La filmographie n'est pas stockee dans l'acteur.
        Elle est calculee en parcourant la liste des films.

afficher_filmographie_realisateur
    Parcourt tous les films.
    Si le pointeur du realisateur du film est le meme que celui recherche,
    le film est affiche.

liberer_films
    Libere chaque noeud Film de la liste chainee.
    Elle ne libere pas les acteurs ni les realisateurs, car ils appartiennent
    a leurs propres listes.


10. Role des fonctions de fichier.c
-----------------------------------

enlever_fin_ligne
    Supprime les caracteres \n et \r a la fin d'une ligne lue depuis le CSV.

texte_en_genre
    Transforme le texte du CSV en valeur de l'enum Genre.
    Exemple : "DRAME" devient DRAME.

charger_acteur
    Recoit les morceaux d'une ligne ACTEUR.
    Cree un Acteur.
    L'ajoute a la liste chainee des acteurs.

charger_realisateur
    Recoit les morceaux d'une ligne REALISATEUR.
    Cree un Realisateur.
    L'ajoute a la liste chainee des realisateurs.

charger_film
    Recoit les morceaux d'une ligne FILM.
    Cherche le realisateur par son nom.
    Cree le film.
    Cherche chaque acteur par son nom.
    Ajoute les acteurs au film.
    Ajoute le film a la liste chainee des films.

decouper_ligne
    Utilise strtok pour separer une ligne CSV avec le separateur ';'.
    Elle remplit un tableau de chaines appele morceaux.

charger_base
    Ouvre donnees.csv avec fopen.
    Lit le fichier ligne par ligne avec fgets.
    Ignore les lignes vides et les commentaires qui commencent par #.
    Decoupe chaque ligne.
    Selon le premier mot, appelle la bonne fonction :
        ACTEUR       -> charger_acteur
        REALISATEUR -> charger_realisateur
        FILM        -> charger_film
        CINEMA      -> ajouter_cinema
        ROUTE       -> ajouter_route
    Ferme le fichier avec fclose.

sauvegarder_base
    Ouvre donnees.csv en ecriture.
    Reecrit les acteurs, realisateurs, films, cinemas et routes.
    Elle est appelee apres l'ajout ou la suppression d'un film.


11. Role des fonctions de cinema.c
----------------------------------

copier_chaine
    Copie une chaine dans un tableau de caracteres.
    Si la chaine source est NULL, elle utilise une chaine vide.

compter_cinemas
    Compte le nombre de cinemas dans le graphe.
    Cette valeur est utile pour allouer les tableaux de Dijkstra.

index_cinema
    Cherche la position d'un cinema dans un tableau de pointeurs.
    Dijkstra travaille avec des indices, donc cette fonction fait le lien
    entre un pointeur Cinema et son indice.

route_existe
    Verifie si une route existe deja entre deux cinemas.
    Cela evite d'ajouter deux fois la meme route.

ajouter_arete
    Ajoute une route orientee entre deux cinemas.
    Elle est static car elle n'est utilisee qu'a l'interieur de cinema.c.

initialiser_graphe
    Met graphe->cinemas a NULL.

creer_cinema
    Alloue un Cinema avec malloc.
    Initialise l'id, le nom, la liste des voisins et le pointeur suivant.
    Retourne un pointeur vers le cinema cree.

ajouter_cinema
    Ajoute un cinema en tete de la liste chainee du graphe.

trouver_cinema_par_id
    Parcourt la liste des cinemas.
    Retourne le cinema dont l'id correspond.

ajouter_route
    Cherche les deux cinemas par id.
    Ajoute une route dans les deux sens :
        depart -> arrivee
        arrivee -> depart
    Le graphe est donc non oriente.

lister_cinemas
    Affiche chaque cinema et ses voisins.

optimiser_trajet
    Applique l'algorithme de Dijkstra.
    But : trouver le chemin le plus court entre deux cinemas.

    Les tableaux utilises :
        tableau   : contient les pointeurs vers les cinemas ;
        distance  : meilleure distance connue depuis le depart ;
        visite    : indique si le cinema est deja traite ;
        precedent : permet de reconstruire le chemin final.

    Principe :
        1. Toutes les distances valent INF sauf le depart qui vaut 0.
        2. On choisit le cinema non visite avec la plus petite distance.
        3. On regarde ses voisins.
        4. Si passer par ce cinema donne une distance plus courte, on met a jour.
        5. A la fin, on reconstruit le chemin avec le tableau precedent.

liberer_graphe
    Libere toutes les routes de chaque cinema.
    Libere ensuite chaque cinema.


12. Pourquoi les donnees ne sont pas dupliquees ?
-------------------------------------------------

Dans un mauvais modele, chaque film pourrait contenir directement toutes les
informations de son realisateur :

    nom, prenom, date de naissance, nationalite

Probleme :
    Si David Fincher realise 3 films, ses informations seraient copiees 3 fois.
    Si on corrige sa nationalite ou sa date de naissance, il faudrait corriger
    plusieurs films.

Dans ce projet :

    Realisateur *realisateur;
    Acteur *acteurs[NB_ACTEURS_MAX];

Le film garde seulement des pointeurs vers des acteurs/realisateurs existants.
Les informations sont stockees une seule fois dans les listes de personnes.


13. Pourquoi utiliser des listes chainees ?
------------------------------------------

Le sujet dit que les listes chainees sont plus judicieuses pour optimiser
l'allocation memoire.

Avantages :

- pas besoin de connaitre le nombre de films a l'avance ;
- on peut ajouter dynamiquement des elements avec malloc ;
- on alloue seulement ce dont on a besoin.

Inconvenient :

- pour chercher un element, il faut parcourir la liste depuis le debut.

Pour un petit projet etudiant, c'est largement suffisant.


14. Pourquoi utiliser des fichiers .h et .c separes ?
-----------------------------------------------------

Les fichiers .h contiennent :

- les structures ;
- les prototypes des fonctions ;
- les constantes utiles.

Les fichiers .c contiennent :

- l'implementation des fonctions.

Avantages :

- le code est mieux organise ;
- main.c reste plus court ;
- chaque module a une responsabilite claire ;
- le compilateur connait les prototypes avant les appels.


15. Questions possibles et reponses
-----------------------------------

Question 1 :
    Est-ce que votre programme est une vraie base de donnees ?

Reponse :
    Non, ce n'est pas une base SQL. C'est une base de donnees simulee en C.
    Les donnees sont stockees dans des structures et listes chainees en memoire.
    Au demarrage, elles sont chargees depuis un fichier CSV.


Question 2 :
    Ou sont stockees les donnees ?

Reponse :
    Sur disque, elles sont dans donnees.csv.
    En memoire, elles sont dans trois listes principales :
        NoeudActeur *acteurs;
        NoeudRealisateur *realisateurs;
        Film *films;
    Les cinemas sont dans :
        GrapheCinema graphe;


Question 3 :
    Comment evitez-vous la duplication des donnees ?

Reponse :
    Un film ne contient pas une copie complete des acteurs et du realisateur.
    Il contient des pointeurs :
        Realisateur *realisateur;
        Acteur *acteurs[NB_ACTEURS_MAX];
    Ainsi, un acteur ou realisateur est stocke une seule fois.


Question 4 :
    Pourquoi la filmographie n'est-elle pas stockee dans l'acteur ?

Reponse :
    Pour eviter la redondance.
    La relation film -> acteur existe deja dans chaque film.
    Pour obtenir la filmographie, il suffit de parcourir tous les films et de
    regarder si l'acteur apparait dans le tableau d'acteurs du film.


Question 5 :
    Que se passe-t-il quand on cherche la filmographie de David Fincher ?

Reponse :
    Le programme cherche d'abord le realisateur Fincher dans la liste des
    realisateurs.
    Ensuite, il parcourt la liste des films.
    Pour chaque film, il compare le pointeur film->realisateur avec le pointeur
    du realisateur cherche.
    Si les pointeurs sont egaux, le film appartient a sa filmographie.


Question 6 :
    Pourquoi comparez-vous parfois des pointeurs au lieu de comparer des noms ?

Reponse :
    Parce que les films referencent directement les acteurs/realisateurs
    existants.
    Si deux pointeurs sont egaux, cela veut dire qu'ils designent exactement
    la meme personne en memoire.
    C'est plus logique dans ce modele que de comparer toutes les informations.


Question 7 :
    Pourquoi ajouter les acteurs et realisateurs avant les films dans le CSV ?

Reponse :
    Parce qu'un film doit pointer vers des acteurs et un realisateur existants.
    Donc quand on lit une ligne FILM, les personnes doivent deja etre presentes
    dans les listes.


Question 8 :
    Pourquoi utiliser malloc ?

Reponse :
    malloc permet d'allouer dynamiquement les noeuds des listes chainees.
    Comme le nombre d'acteurs, realisateurs, films ou cinemas peut changer
    selon le fichier CSV, on ne fixe pas une taille definitive dans le code.


Question 9 :
    Ou liberez-vous la memoire ?

Reponse :
    A la fin de main, avec :
        liberer_films(films);
        liberer_acteurs(acteurs);
        liberer_realisateurs(realisateurs);
        liberer_graphe(&graphe);
    Chaque fonction parcourt sa liste et appelle free sur les noeuds alloues.


Question 10 :
    Pourquoi liberer les films ne libere pas les acteurs ?

Reponse :
    Parce que les acteurs ne sont pas possedes par les films.
    Les films contiennent seulement des pointeurs vers les acteurs.
    Les acteurs sont stockes dans leur propre liste, donc ils sont liberes par
    liberer_acteurs.


Question 11 :
    Pourquoi avoir un enum pour les genres ?

Reponse :
    Un enum limite les valeurs possibles.
    Cela correspond au sujet, qui donne une liste de genres imposes.
    C'est plus propre qu'une simple chaine de caracteres libre.


Question 12 :
    Comment la recherche d'un film fonctionne-t-elle ?

Reponse :
    Le programme parcourt la liste chainee des films.
    Pour chaque film, il compare le titre avec le titre saisi.
    La comparaison ignore les majuscules/minuscules grace a comparer_sans_casse.


Question 13 :
    Pourquoi utiliser fgets au lieu de scanf pour le menu ?

Reponse :
    fgets lit toute la ligne saisie.
    Ensuite, sscanf convertit en entier.
    Cela evite des problemes de retour a la ligne restant dans le buffer clavier.


Question 14 :
    Comment fonctionne le chargement CSV ?

Reponse :
    charger_base ouvre donnees.csv avec fopen.
    Elle lit chaque ligne avec fgets.
    Elle ignore les commentaires et lignes vides.
    Elle decoupe la ligne avec strtok en utilisant ';' comme separateur.
    Selon le premier champ, elle cree un acteur, un realisateur, un film, un
    cinema ou une route.


Question 15 :
    Quelles sont les limites du format CSV ?

Reponse :
    Le format est volontairement simple.
    Les champs ne doivent pas contenir le caractere ';'.
    Les films doivent etre apres les acteurs/realisateurs.
    Les routes doivent etre apres les cinemas.
    Pour un TP, ce format est suffisant et facile a expliquer.


Question 16 :
    Comment est represente le graphe de cinemas ?

Reponse :
    Le graphe contient une liste chainee de cinemas.
    Chaque cinema contient une liste chainee de routes vers ses voisins.
    Une route est representee par une structure Arete avec une destination et
    une distance.


Question 17 :
    Pourquoi ajoutez-vous une route dans les deux sens ?

Reponse :
    Parce que les routes entre cinemas sont considerees comme non orientees.
    Si on peut aller de ESIEE a Noisy Centre, on peut aussi revenir de Noisy
    Centre a ESIEE avec la meme distance.


Question 18 :
    Quel algorithme utilisez-vous pour le trajet ?

Reponse :
    J'utilise Dijkstra.
    C'est coherent avec OSPF, car OSPF utilise un algorithme de plus court
    chemin pour calculer les meilleures routes.


Question 19 :
    Expliquez Dijkstra simplement.

Reponse :
    On part du cinema de depart avec une distance de 0.
    Tous les autres cinemas ont une distance infinie.
    A chaque etape, on choisit le cinema non visite avec la plus petite distance.
    On regarde ses voisins et on met a jour leur distance si on trouve un chemin
    plus court.
    A la fin, on reconstruit le chemin grace au tableau precedent.


Question 20 :
    A quoi sert le tableau precedent dans Dijkstra ?

Reponse :
    Il permet de reconstruire le chemin final.
    Pour chaque cinema, precedent indique par quel cinema on est passe pour
    obtenir la meilleure distance.


Question 21 :
    Pourquoi utilisez-vous INF ?

Reponse :
    INF represente une distance initialement inconnue ou infinie.
    Au debut, seuls les cinemas non encore atteints ont cette distance.


Question 22 :
    Que se passe-t-il si un cinema n'existe pas ?

Reponse :
    optimiser_trajet cherche les cinemas par id.
    Si le depart ou l'arrivee est introuvable, la fonction affiche un message
    d'erreur et s'arrete proprement.


Question 23 :
    Pourquoi certaines fonctions sont static ?

Reponse :
    static limite la visibilite de la fonction au fichier .c.
    Par exemple, copier ou decouper_ligne sont des fonctions internes.
    Elles n'ont pas besoin d'etre appelees depuis les autres fichiers.


Question 24 :
    Pourquoi utiliser des fichiers .h ?

Reponse :
    Les .h donnent les prototypes et les structures visibles par les autres
    fichiers.
    Cela permet de separer l'interface du module et son implementation.


Question 25 :
    Comment compiler le projet ?

Reponse :
    Avec gcc :
        gcc -Wall main.c personne.c film.c cinema.c fichier.c -o CineEsiee.exe

    Version stricte :
        gcc -Wall -Wextra -pedantic -std=c99 main.c personne.c film.c cinema.c fichier.c -o CineEsiee.exe


Question 26 :
    Pourquoi votre programme doit-il avoir 0 warning ?

Reponse :
    Les warnings signalent souvent des erreurs potentielles :
    mauvais prototype, variable inutilisee, conversion dangereuse, etc.
    Un programme avec 0 warning est plus propre et plus fiable.


Question 27 :
    Que se passe-t-il si donnees.csv est absent ?

Reponse :
    charger_base retourne 0.
    main affiche un message :
        Impossible de charger donnees.csv.
    Puis le programme s'arrete avec return 1.


Question 28 :
    Pourquoi le menu contient plus que les 4 options obligatoires ?

Reponse :
    Les 4 options obligatoires sont bien presentes :
        lister les films, lister les realisateurs, rechercher, quitter.
    Les autres options sont des extensions utiles :
        lister acteurs, trajet, filmographie.


Question 29 :
    Quel est le role de nb_acteurs dans Film ?

Reponse :
    nb_acteurs indique combien de cases du tableau acteurs sont utilisees.
    Cela evite de parcourir les cases vides.


Question 30 :
    Pourquoi NB_ACTEURS_MAX vaut 3 ?

Reponse :
    Le sujet demande de stocker les acteurs principaux de 1 a 3.
    Donc le tableau de pointeurs d'acteurs a une taille maximale de 3.

17. Points faibles a connaitre
------------------------------

Il vaut mieux connaitre les limites du projet si le professeur les demande.

- Le CSV est simple : il ne gere pas les champs avec des ';' a l'interieur.
- La recherche d'acteur/realisateur se fait par nom exact.
- Le programme modifie le CSV pour les films, mais pas pour creer de nouveaux
  acteurs ou realisateurs.
- Le trajet calcule le plus court chemin entre deux cinemas, pas une tournee
  complete avec plusieurs livraisons.
- Les listes chainees sont simples, mais la recherche est lineaire.

Ces limites sont acceptables pour un projet C etudiant, car l'objectif est de
montrer les structures, les listes chainees, les pointeurs et les fichiers.


18. Resume en 1 minute
----------------------

"Mon projet CineEsiee simule une base de donnees de films en C. Les donnees
sont lues depuis donnees.csv. En memoire, j'utilise des listes chainees pour les
acteurs, les realisateurs et les films. Un film contient un pointeur vers son
realisateur et un tableau de pointeurs vers ses acteurs principaux, donc les
informations des personnes ne sont pas dupliquees. Le menu permet de lister les
films, lister les realisateurs, rechercher un film, ajouter ou supprimer un
film, afficher des filmographies et calculer un trajet entre cinemas. Quand un
film est ajoute ou supprime, donnees.csv est reecrit. La partie trajet utilise
un graphe pondere et l'algorithme de Dijkstra, inspire du fonctionnement
d'OSPF. A la fin du programme, toute la memoire allouee dynamiquement est
liberee."
